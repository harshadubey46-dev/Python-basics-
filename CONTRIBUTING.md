# Contributing to Python Basics & DSA Repository

Thank you for your interest in contributing! 💖

## How to Contribute

1. **Fork** the repository
2. **Clone** your fork locally
   ```bash
   git clone https://github.com/YOUR-USERNAME/Python-basics-.git
   ```
3. **Create a new branch** for your feature
   ```bash
   git checkout -b feature/your-feature-name
   ```
4. **Make your changes** with clear comments and documentation
5. **Commit** with meaningful messages
   ```bash
   git commit -m "Add: descriptive message about your changes"
   ```
6. **Push** to your fork
   ```bash
   git push origin feature/your-feature-name
   ```
7. **Create a Pull Request** on the main repository

## Guidelines

### Code Quality
- ✨ Write clean, readable code
- 💬 Add comments explaining logic (especially for DSA programs)
- ⚡ Include time and space complexity analysis
- 🧪 Test your code before submitting

### Commit Messages
- Use clear, descriptive messages
- Format: `Add:`, `Fix:`, `Update:`, `Refactor:` prefix
- Example: `Add: Stack implementation with complexity analysis`

### File Naming
- Use snake_case for filenames: `bubble_sort.py`
- Organize in appropriate topic folders
- Include problem description in comments

### Documentation
Every program should include:
```python
"""
Program Name: [Name]
Description: [What it does]
Time Complexity: O(n) or as applicable
Space Complexity: O(n) or as applicable
Author: [Your name]
Date: [Date added]
"""
```
